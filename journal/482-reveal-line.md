## 2026-05-07T18:12:00-04:00 — push 3: hosted CI e2e fix (inline spans, no transform)

**Pushed:** fix(reveal-line): drop inline-block + transform to fix hosted-CI layout drift
**Why:** Hosted CI e2e on PR #506 failed `demo-challenge-mobile` visual-regression with a 29px page-height delta (~one text line). Local visual-regression passed 57/57 against the same baselines, so it looked at first like the documented hosted-vs-local Playwright render-delta — but the artifact diff PNG showed clear ghosting on the framing line area and every modifier UI row below it, which is "real layout shift," not "sub-pixel rounding." Root cause: each word in `RevealLine` rendered as `<span class="inline-block">`, which has whitespace ambiguity between adjacent boxes that local and hosted Chromium resolved differently — local matched the plain-text baseline (1147px), hosted produced 1118px. Fix: switched to plain `<span>` (default `inline` display) and dropped the `translateY(2px)` transform from the keyframe (transforms don't apply to inline elements anyway, and opacity is the universally-supported animation channel). The staggered word-reveal still reads as speech without the slide-up. Visual-regression `--update-snapshots` ran with **zero baseline diffs**, confirming inline spans render identically to plain text.
