# Portrait asset licensing

The 10 character portraits shipped under `public/portraits/<character>/`
are AI-generated digital paintings (initial drop, 2026-05-07). The
source PNGs live outside the repository (in a gitignored
`greek_gods/NoBG/GreekGods/` staging directory) at native
3840×2160 (16:9, 4K). Runtime WebPs are derived from those sources
via the rebuild recipe at the bottom of this file.

These are **placeholder-grade illustrations** — usable for the
encounter-theatricality slice (#474) while higher-fidelity commissioned
art is sourced. Final assets land at the same paths in
`public/portraits/<character>/{large,small}.webp` and the existing
wiring picks them up without code change.

## Per-file inventory

For each character, two runtime files are shipped:

| File | Dimensions | Use |
|---|---|---|
| `public/portraits/<character>/large.webp` | 1024×576 | High-DPI runtime — `AvatarPortrait` at the re-skinned prep stage centre, Codex pages, marketing surfaces |
| `public/portraits/<character>/small.webp` | 480×270 | Low-DPI / mobile / smaller crops |

All files are 16:9 horizontal with a centred figure on transparent
background. The circular `AvatarPortrait` frame uses `object-fit:
cover` + `object-position` to crop around the figure — adjust
per-character `object-position` if the head is off-centre.

## Rebuilding runtime WebPs from source PNGs

```bash
SRC=greek_gods/NoBG/GreekGods
DST=public/portraits

for src in "$SRC"/*.png; do
  name=$(basename "$src" .png)
  char=$(echo "$name" | tr '[:upper:]' '[:lower:]')
  outdir="$DST/$char"
  mkdir -p "$outdir"

  ffmpeg -y -loglevel error -i "$src" -vf "scale=1024:-1:flags=lanczos" \
    -c:v libwebp -compression_level 6 -quality 80 -preset picture "$outdir/large.webp"

  ffmpeg -y -loglevel error -i "$src" -vf "scale=480:-1:flags=lanczos" \
    -c:v libwebp -compression_level 6 -quality 80 -preset picture "$outdir/small.webp"
done
```

Requires `ffmpeg` ≥ 5 with `libwebp`. The script is deterministic at
quality=80; running it twice on the same machine produces byte-identical
output, so a developer who replaces a source PNG can regenerate without
unrelated diff churn.

## Source staging

Source PNGs live at `greek_gods/NoBG/GreekGods/<Character>.png`
(native 3840×2160) and are **gitignored** — they are too large to
commit and only needed at rebuild time. Replacement workflow:

1. Drop a new high-resolution PNG into `greek_gods/NoBG/GreekGods/<Character>.png`.
2. Run the rebuild snippet above (or just the relevant `ffmpeg` lines for the one character).
3. Update this file with the new source / artist / licence.
4. Commit only `public/portraits/<character>/*.webp` (and this file).
